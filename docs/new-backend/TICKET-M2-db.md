# M-2 — `@nebula-chat/db` (Drizzle ORM, replaces Prisma)

## Ticket metadata

| Field          | Value                                                             |
| -------------- | ----------------------------------------------------------------- |
| **ID**         | M-2                                                               |
| **Package**    | `libs/db` → published as `@nebula-chat/db`                        |
| **Depends on** | Nothing — fully independent lib ticket                            |
| **Blocks**     | Nothing (apps/server consumes it, but can use Prisma temporarily) |
| **Standalone** | Yes                                                               |

## Objective

Create `libs/db` as a standalone versioned package containing the Drizzle client, all database table schemas, and migration tooling. `apps/server` imports `@nebula-chat/db` instead of using Prisma directly. Prisma is fully removed after this ticket.

## Acceptance criteria

- [ ] `libs/db/` exists with correct `package.json`
- [ ] `libs/db/src/schema.ts` defines all tables (users, conversations, messages)
- [ ] `libs/db/src/client.ts` exports `createDbClient` factory
- [ ] `libs/db/src/index.ts` exports everything
- [ ] `drizzle.config.ts` exists and points to the correct schema and output paths
- [ ] `pnpm --filter @nebula-chat/db db:generate` generates migrations without error
- [ ] `pnpm --filter @nebula-chat/db db:push` applies schema to local Postgres without error
- [ ] `apps/server` has `@nebula-chat/db: "workspace:*"` in dependencies
- [ ] A Fastify plugin in `apps/server/src/plugins/db.plugin.ts` decorates the app with `db`
- [ ] Prisma and all `@prisma/*` packages are removed from `apps/server/package.json`

### File: `libs/db/src/schema.ts`

```ts
import { pgTable, uuid, text, timestamp, integer, boolean, index } from 'drizzle-orm/pg-core';

export const users = pgTable('users', {
  id: uuid('id').primaryKey().defaultRandom(),
  email: text('email').notNull().unique(),
  passwordHash: text('password_hash'),
  displayName: text('display_name'),
  avatarUrl: text('avatar_url'),
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').defaultNow().notNull(),
});

export const conversations = pgTable(
  'conversations',
  {
    id: uuid('id').primaryKey().defaultRandom(),
    userId: uuid('user_id')
      .references(() => users.id, { onDelete: 'cascade' })
      .notNull(),
    title: text('title').notNull().default('New conversation'),
    model: text('model').notNull().default('gpt-4o-mini'),
    systemPrompt: text('system_prompt'),
    createdAt: timestamp('created_at').defaultNow().notNull(),
    updatedAt: timestamp('updated_at').defaultNow().notNull(),
  },
  (t) => [index('conv_user_idx').on(t.userId)],
);

export const messages = pgTable(
  'messages',
  {
    id: uuid('id').primaryKey().defaultRandom(),
    conversationId: uuid('conversation_id')
      .references(() => conversations.id, { onDelete: 'cascade' })
      .notNull(),
    role: text('role', { enum: ['user', 'assistant', 'system'] }).notNull(),
    content: text('content').notNull(),
    tokenCount: integer('token_count'),
    cached: boolean('cached').default(false),
    createdAt: timestamp('created_at').defaultNow().notNull(),
  },
  (t) => [index('msg_conv_idx').on(t.conversationId)],
);
```

### File: `libs/db/src/client.ts`

```ts
import { drizzle } from 'drizzle-orm/node-postgres';
import { Pool } from 'pg';
import * as schema from './schema';

export interface DbConfig {
  connectionString: string;
  maxConnections?: number;
}

export function createDbClient(config: DbConfig) {
  const pool = new Pool({
    connectionString: config.connectionString,
    max: config.maxConnections ?? 10,
  });

  return drizzle(pool, { schema });
}

export type DbClient = ReturnType<typeof createDbClient>;
```

## Consuming in `apps/server`

### 1. Add workspace dependency

In `apps/server/package.json`:

```json
{
  "dependencies": {
    "@nebula-chat/db": "workspace:*"
  }
}
```

Run `pnpm install` to link the workspace package.

### 2. Fastify plugin

```ts
// apps/server/src/plugins/db.plugin.ts
import fp from 'fastify-plugin';
import { createDbClient } from '@nebula-chat/db';
import { env } from '../env';

export default fp(async (app) => {
  const db = createDbClient({ connectionString: env.DATABASE_URL });
  app.decorate('db', db);
});

// Extend Fastify types so `app.db` is typed everywhere
declare module 'fastify' {
  interface FastifyInstance {
    db: import('@nebula-chat/db').DbClient;
  }
}
```

Register in `server.ts`:

```ts
await app.register(import('./plugins/db.plugin'));
```

### 3. Using in a service

```ts
// apps/server/src/modules/chat/chat.service.ts
import { eq, desc } from 'drizzle-orm';
import { conversations, messages } from '@nebula-chat/db';
import type { DbClient } from '@nebula-chat/db';

export async function getConversation(db: DbClient, id: string) {
  return db.query.conversations.findFirst({
    where: eq(conversations.id, id),
    with: { messages: { orderBy: desc(messages.createdAt) } },
  });
}
```

## Removing Prisma

After this ticket is complete and all Prisma usage is replaced:

1. Remove from `apps/server/package.json` dependencies:
   - `@prisma/client`
   - `@prisma/adapter-pg`

2. Remove from `apps/server/package.json` devDependencies:
   - `prisma`

3. Remove from root `package.json` `pnpm.onlyBuiltDependencies`:
   - `@prisma/engines`
   - `prisma`

4. Delete `apps/server/prisma/` directory

5. Remove all Prisma scripts from `apps/server/package.json`:
   - `prisma:generate`, `prisma:migrate`, `prisma:deploy`, `prisma:studio`, `prisma:validate`

## Migration workflow (running for the first time)

```bash
# Start local Postgres (see CI-4)
pnpm dev:docker

# Set DATABASE_URL for the db package
export DATABASE_URL=<your_database_url>

# Push schema directly (dev only — no migration files needed initially)
pnpm --filter @nebula-chat/db db:push

# Generate migration files (for production deploys)
pnpm --filter @nebula-chat/db db:generate

# Apply migrations (production)
pnpm --filter @nebula-chat/db db:migrate
```
